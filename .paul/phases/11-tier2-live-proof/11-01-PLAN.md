## Goal
Add an opt-in, network-gated live integration proof that sends the real `buildTier2Request` output to the running local `mlx_vlm.server` endpoint and verifies `parseTier2Answer` reads a real answer from Qwen3-VL. If the live proof exposes a tier-2 adapter wire-shape mismatch, fix that mismatch without adding model-specific branches.

## Purpose
v0.1 shipped a model-agnostic tier-2 adapter (`src/watch/tier2.ts`) with deterministic fake-`fetch` tests, and Phase 10 stood up a real local Qwen3-VL endpoint. The remaining gap is that the production request-builder/response-parser path has still not touched the real endpoint. Phase 11 closes that gap while preserving the project constraint that tier 2 is only `baseURL` + `model id`, not a per-model code fork.
