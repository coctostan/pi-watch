<objective>
## Goal
Stand up a local OpenAI-compatible vision endpoint ŌĆö `mlx_vlm.server` serving a Qwen3-VL checkpoint ŌĆö on this Mac (Apple M4 Pro, 48 GB), prove it answers a frames-as-images `/v1/chat/completions` request, and capture the exact, reproducible steps as a committed runbook (`docs/TIER2-SETUP.md`) including the `WATCH_TIER2_*` values that wire the `watch` tool to it.

## Purpose
v0.1 shipped the full tier-2 adapter (`buildTier2Request` / `parseTier2Answer` / `createTier2Runner`) but it has never exchanged a byte with a real model ŌĆö every test injects a fake `fetch`. Tier 2 is the local native-video tier that makes pi-watch worth it (DESIGN ┬¦4, Core Value: local-first, model-agnostic). Nothing downstream (Phase 11 live wire-shape proof, Phase 12 diagnostics) can be validated until a real endpoint exists. This phase removes the environment blocker the user explicitly asked for help with, and turns the one-off feasibility spike (DESIGN ┬¦5 #2: Qwen3-VL answered `red -> green -> blue` in 3.3s, warm RSS 7.6 GB) into a documented, repeatable local setup.

## Output
- A running `mlx_vlm.server` on localhost serving a Qwen3-VL checkpoint, reachable at an OpenAI-compatible `/v1/chat/completions` URL.
- A pinned Python environment (via `uv`) with `mlx_vlm` importable, isolated from the system Python 3.14.6 (too new for `mlx_vlm` wheels).
- `docs/TIER2-SETUP.md` ŌĆö the reproducible runbook: prerequisites, exact commands, chosen model + quant, the smoke-test request, and the `WATCH_TIER2_BASE_URL` / `WATCH_TIER2_MODEL` values to export.
</objective>
