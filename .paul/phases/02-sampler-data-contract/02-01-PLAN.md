<objective>
## Goal
Establish the project's first production TypeScript surface and define the **"watched frame set" data contract** — the in-memory type the sampler produces and every tier (1/2/3) consumes — plus a pure serializer to the OpenAI `content[]` wire shape. Includes minimal project scaffolding (package.json, tsconfig, test runner) since no production tooling exists yet.

## Purpose
This contract is **the load-bearing seam every tier plugs into** (DESIGN.md §6, "design before coding"). Both proven spikes already tell us the downstream target: *ordered base64 image blocks + interleaved text in OpenAI `content[]` shape* (DESIGN.md §5). Getting the in-memory representation right now — on one shared timeline — is what lets local-vs-hosted and which-model become config, not code forks (PROJECT.md core decision).

## Output
- A runnable TS project skeleton (build + typecheck + test) with one approved dependency set.
- `src/contract/` exporting the `WatchedFrameSet` type family and a pure `toOpenAIContent()` serializer.
- A deterministic test suite asserting contract invariants and serialization shape against fixtures.
</objective>

<task type="auto">
  <name>Task 2: Define the WatchedFrameSet contract types + runtime schema</name>
  <files>src/contract/watched-frame-set.ts, src/contract/index.ts</files>
  <action>
    Define the in-memory data contract on ONE shared timeline (DESIGN.md §6, PRD line 42). Use the approved validation lib so each shape yields both a static type and a runtime validator.
    Required shapes:
    - ResolutionTier: "low" | "high".
    - FrameOrigin: "scene-cut" | "backfill".
    - TranscriptSource: "captions" | "whisper".
    - WatchedFrame: { index: number; tMs: number (>=0, canonical offset); timestamp: string (mm:ss display, derived from tMs); imageBase64: string; mediaType: "image/png" | "image/jpeg"; resolution: ResolutionTier; origin: FrameOrigin }.
    - TranscriptSegment: { startMs: number (>=0); endMs: number (>= startMs); text: string; source: TranscriptSource }.
    - SourceMetadata: { ref: string; durationMs: number (>=0); fpsSampled: number (>0); frameCount: number (>=0); transcriptSource: TranscriptSource | "none" }.
    - WatchedFrameSet: { source: SourceMetadata; frames: WatchedFrame[] (ordered by tMs ascending); transcript: TranscriptSegment[] (ordered by startMs ascending) }.
    Provide a pure validator (e.g. `validateWatchedFrameSet(value): { ok: true; value } | { ok: false; errors: string[] }`) that enforces: frames ordered by tMs, transcript ordered by startMs, frameCount === frames.length, no negative times, endMs>=startMs per segment. Keep it a pure function (no I/O). Re-export the public surface from index.ts.
    Avoid: storing raw video bytes, file handles, or any tier-specific fields — this is the neutral contract only. Do NOT embed OpenAI-specific shapes here (that belongs in serialize.ts).
  </action>
  <verify>`npm run typecheck` exits 0; a quick `node --import tsx -e` or test asserts the validator rejects an out-of-order/negative-time fixture.</verify>
  <done>AC-2 + AC-4 satisfied: contract compiles, validates well-formed values, rejects malformed ones, and covers all DESIGN §6 / PRD fields.</done>
</task>
