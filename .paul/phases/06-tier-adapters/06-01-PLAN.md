<objective>

## Goal
Make the tier-runner seam *real* and land the cheapest tier. Two coupled outcomes:

1. **Async seam.** Promote `TierRunner` from synchronous (`(args) => TierResult | null`) to
   asynchronous (`(args) => Promise<TierResult | null>`), make `walkTierChain` async, and
   `await` it at the extension boundary. This is the load-bearing prerequisite for tier 2
   (an OpenAI-compatible network adapter, Plan 06-02) and revises the Phase-5 carry note
   ("sync signature / no extension.ts change needed") — network I/O is unavoidably async.
2. **Tier 1 (transcript).** Replace the `tier1Runner` null stub with a real adapter: when a
   usable transcript exists, hand the transcript text back to the orchestrator model as a
   tier-1 `TierResult` (the cheapest, model-agnostic answering path — no frames, no external
   model call); otherwise return `null` to escalate. Tier 3 (frames-into-context) stays the
   total terminal fallback.

## Purpose
This is the first half of Phase 6 (tier adapters). It establishes the async seam every real
adapter needs and proves it end-to-end with the simplest real tier, so Plan 06-02 can drop in
the tier-2 HTTP adapter on a stable surface without re-touching `walkTierChain` or
`extension.ts`. It honors the project's core value (PROJECT.md): "cheapest path that works,
local-first, model-agnostic" — tier 1 answers spoken-content questions with zero external
dependency.

<acceptance_criteria>

## AC-1: The tier seam is asynchronous and preserves existing walk semantics
```gherkin
Given the watch tier seam
When walkTierChain is called with an ordered tier chain
Then it returns a Promise<TierResult>, awaiting each runner in order
And it returns the first non-null result and does not invoke later tiers
And it rejects/throws when no tier in the chain produces a result
And src/watch/extension.ts awaits walkTierChain at the effect boundary
And the full test suite and typecheck/build pass
```

## AC-2: Tier 1 escalates when there is no usable transcript
```gherkin
Given a WatchedFrameSet whose transcript is empty or whose transcriptSource is "none"
When tier1Runner runs (e.g. as part of a chain)
Then it resolves to null
And walkTierChain escalates past tier 1 to the next tier in the chain
```

## AC-3: Tier 1 answers from the transcript when one is present
```gherkin
Given a WatchedFrameSet with transcript segments and transcriptSource "captions" or "whisper"
And a spoken-content routing decision with chain [1, 2, 3]
When walkTierChain runs
Then it resolves at tier 1 with a TierResult whose content is the transcript handed to the orchestrator
And tiers 2 and 3 are not invoked
And the content includes the question and the transcript's mm:ss-labelled lines
```
