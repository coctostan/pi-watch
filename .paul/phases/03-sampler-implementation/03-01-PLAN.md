<objective>
## Goal
Build the **deterministic, pure core of the sampler**: the frame-selection algorithm
(scene-cuts + uniform backfill + budget cap) and the assembly of a validated
`WatchedFrameSet` (including transcript merge onto the shared timeline). No ffmpeg,
no I/O, no model calls — every function is pure input→output and fully unit-tested
against deterministic fixtures.

## Purpose
The sampler is "the heart of it" (DESIGN.md §3) and the one thing **we own** —
local-vs-hosted becomes config because every tier consumes the `WatchedFrameSet` this
core produces (PROJECT.md core value). The selection/budget logic is the load-bearing
*decision* layer: it decides which moments of a video the model ever sees and enforces
the frame budget that is our guardrail against token blowup (PRD Risks). Isolating it
as a pure core (the ffmpeg effect boundary lands in 03-02) makes it the kind of
"pure input-to-output business logic" the PRD's Testing Strategy says to assert
deterministically, independent of any probabilistic model output.

## Output
- `src/sampler/select-frames.ts` — pure `selectFrameTimes()` returning ordered,
  budget-capped frame times tagged `scene-cut` | `backfill`.
- `src/sampler/assemble.ts` — pure `mergeTranscript()` + `assembleWatchedFrameSet()`
  + `formatTimestamp()`, producing a value that passes `validateWatchedFrameSet`.
- `src/sampler/index.ts` — public surface for the sampler core.
- `test/sampler/select-frames.test.ts`, `test/sampler/assemble.test.ts` —
  deterministic unit tests (RED-first).
</objective>

<acceptance_criteria>

## AC-1: Scene cuts are selected as frames within budget
```gherkin
Given a list of scene-change offsets within a clip duration and a frame budget B
When selectFrameTimes is called and the scene-cut count <= B
Then every scene cut is returned as a frame with origin "scene-cut"
And the result is ordered by tMs ascending with no duplicate offsets
And every tMs is within [0, durationMs)
```

## AC-2: Uniform backfill covers gaps when scene cuts are sparse
```gherkin
Given fewer scene cuts than the frame budget B over a long duration
When selectFrameTimes is called
Then additional frames are inserted with origin "backfill" to approach B
And backfill frames fall in the largest uncovered gaps (no clustering)
And the merged result remains ordered by tMs ascending with no duplicate offsets
```

## AC-3: The frame budget is always enforced
```gherkin
Given any combination of scene cuts, duration, and a budget B (including B greater
  than, equal to, and less than the scene-cut count, plus B of 0 and 1)
When selectFrameTimes is called
Then the returned frame count never exceeds B
And when scene cuts exceed B they are uniformly subsampled (preserving temporal
  spread, deterministically), never simply truncated to the first B
```

## AC-4: Assembly produces a valid WatchedFrameSet on one timeline
```gherkin
Given selected frame times with decoded image payloads, a transcript, and source metadata
When assembleWatchedFrameSet is called
Then the returned value passes validateWatchedFrameSet (ok: true)
And each frame has a sequential zero-based index, an mm:ss timestamp derived from its
  tMs, and the supplied resolution tier
And source.frameCount equals frames.length and source.transcriptSource reflects the
  transcript ("none" when empty)
```

## AC-5: Transcript merges onto the shared timeline deterministically
```gherkin
Given transcript segments that may be unordered or extend past the clip duration
When mergeTranscript is called with the clip duration
Then segments are returned ordered by startMs ascending
And each endMs is clamped to be >= startMs and <= durationMs
And empty/whitespace-only segments are dropped
And the result satisfies the transcript invariants of validateWatchedFrameSet
```

  <action>
    Implement and export from select-frames.ts (pure; no I/O, no mutation of inputs):

    - Types:
        export interface SelectedFrame { tMs: number; origin: FrameOrigin; }
        export interface SelectOptions {
          sceneCutsMs: number[];   // raw scene-change offsets (may be unsorted/dupes)
          durationMs: number;
          budget?: number;         // default 16 (DESIGN §3); must be >= 0 integer
        }
      Import FrameOrigin from "../contract/index.js".

    - export function selectFrameTimes(opts: SelectOptions): SelectedFrame[]
      Algorithm (document each step in a comment, keep deterministic):
        1. Normalize cuts: keep integers in [0, durationMs), dedupe, sort ascending.
        2. budget = Math.max(0, Math.floor(budget ?? 16)). If budget === 0 → return [].
        3. If cuts.length >= budget:
             uniformly subsample cuts to exactly `budget` picks preserving spread
             (e.g. pick index round(i * (cuts.length-1)/(budget-1)) for i in 0..budget-1;
             dedupe resulting indices). All origin "scene-cut". (budget===1 → first cut.)
        4. Else (cuts.length < budget):
             keep all cuts as "scene-cut"; compute remaining = budget - cuts.length;
             generate uniform backfill offsets across [0, durationMs) that land in the
             largest uncovered gaps, excluding offsets already present; mark "backfill";
             take only `remaining` of them.
        5. Merge, sort by tMs ascending, drop duplicate tMs (scene-cut wins on tie),
           and ensure final length <= budget.
      Constraints: never exceed budget; never emit tMs < 0 or >= durationMs; output
      ordered ascending with unique tMs. Avoid Math.random — fully deterministic.
  </action>
