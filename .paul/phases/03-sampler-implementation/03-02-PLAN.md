<objective>
## Goal
Add the **effect boundary** and the **`sample()` entry point** that turn a real video reference into a validated `WatchedFrameSet`. Shell out to `ffprobe` (duration), `ffmpeg` (scene-cut detection + frame decode), and a best-effort transcript fetch, then feed those values into the frozen pure core (`selectFrameTimes` → decode → `assembleWatchedFrameSet`). Prove it end-to-end with a **golden-clip round-trip** synthesized at test time.

## Purpose
This is the seam where DESIGN.md §3's sampler stops being arithmetic and starts watching real video. It is local-first (system `ffmpeg`/`ffprobe`, optional captions/Whisper, never a mandatory cloud or `whisper` dependency) and keeps the load-bearing design promise: **WE own the sampling**; `sample()` is the one surface Phase 4 (router) and Phase 5 (`watch` tool) wrap. The pure core stays pure — every side effect (process spawn, base64 read) lives explicitly at this boundary (AGENTS.md "Pure Core, Explicit Effects").

## Output
- `src/sampler/effects.ts` — `probeDurationMs`, `detectSceneCutsMs`, `decodeFramesAt`, `fetchTranscript`, plus their pure parsers (`parseDurationMs`, `parseSceneCutsMs`).
- `src/sampler/sample.ts` — `sample()` orchestrator wiring effects → pure core → validated `WatchedFrameSet`.
- `src/sampler/index.ts` — extended public surface (`sample`, effect functions, parser exports, option types).
- `test/sampler/effects.test.ts` — direct unit tests for the pure parsers.
- `test/sampler/sample.integration.test.ts` — golden-clip round-trip against an ffmpeg-`lavfi`-synthesized clip.
</objective>

<acceptance_criteria>

## AC-1: ffprobe yields integer duration
```gherkin
Given a readable local video file
When probeDurationMs(ref) runs ffprobe and parses its output
Then it returns a positive integer duration in milliseconds
And parseDurationMs maps ffprobe's seconds (or "N/A") output to that integer (or throws a contextual error)
```

## AC-2: ffmpeg scene detection yields ordered in-range cut offsets
```gherkin
Given a clip with visible scene changes and a known durationMs
When detectSceneCutsMs(ref, durationMs) runs ffmpeg's scene filter and parses its output
Then it returns integer millisecond offsets, ascending, each within [0, durationMs)
And parseSceneCutsMs extracts those offsets from raw ffmpeg showinfo/scene output deterministically
And the raw offsets are safe to hand directly to selectFrameTimes (which re-normalizes them)
```

## AC-3: frame decode is aligned and base64-encoded
```gherkin
Given a ref, a list of frame times in ms, and a ResolutionTier
When decodeFramesAt(ref, timesMs, resolution) extracts those frames
Then it returns FrameImage[] of the same length and order as timesMs
And each FrameImage has a non-empty base64 payload (no "data:" prefix) and a valid MediaType
```

## AC-4: sample() golden-clip round-trip produces a valid WatchedFrameSet
```gherkin
Given a deterministic clip synthesized at test time via ffmpeg lavfi
When sample({ ref, budget }) runs the full effect → pure-core chain
Then validateWatchedFrameSet(result).ok === true
And result.frames.length <= budget, indices are sequential zero-based, timestamps are mm:ss, frames are tMs-ascending
And result.source.frameCount === result.frames.length and source.ref === ref
```

## AC-5: transcript fetch is best-effort and degrades locally
```gherkin
Given a ref for which no captions and no Whisper transcript are available
When sample() calls fetchTranscript and assembles the set
Then sample() still returns a valid WatchedFrameSet
And source.transcriptSource === "none" with an empty transcript array
And no missing-binary / no-network condition throws out of sample()
```

  <name>Task 1: Effect layer — ffprobe/ffmpeg/transcript with pure parsers (src/sampler/effects.ts)</name>
  <files>src/sampler/effects.ts</files>
  <action>
    Create the explicit effect boundary. A small internal helper runs a binary safely; pure parsers are exported separately so they can be unit-tested without spawning anything.

    SAFETY (SETH/REED — non-negotiable):
    - Spawn ONLY via `node:child_process` `execFile` (promisified) or `spawn` with an **argument array**. Never `exec`, never a shell string, never `shell: true`, never string-interpolate `ref` into a command. The `ref` is always a standalone argv element.
    - Every spawn passes a `timeout` (e.g. 60_000 ms) and a generous `maxBuffer`. On non-zero exit, missing binary (`ENOENT`), or timeout, throw an `Error` whose message names the binary and includes captured stderr context.

    Pure parsers (no I/O — exported):
    - `parseDurationMs(probeStdout: string): number` — parse the seconds value ffprobe prints for `format=duration` (e.g. `12.345`), `Math.round(seconds * 1000)`. Throw a contextual error if the value is missing or `N/A`.
    - `parseSceneCutsMs(ffmpegOutput: string, durationMs: number): number[]` — extract frame `pts_time`/`showinfo` timestamps emitted by the scene filter, convert seconds→ms (`Math.round`), keep finite values in `[0, durationMs)`, sort ascending. (selectFrameTimes also normalizes, but return clean data here.)

    Effects (thin wrappers over the helper + a parser):
    - `probeDurationMs(ref: string): Promise<number>` — `ffprobe -v error -show_entries format=duration -of default=noprint_wrappers=1:nokey=1 <ref>` → `parseDurationMs`.
    - `detectSceneCutsMs(ref: string, durationMs: number, threshold = 0.4): Promise<number[]>` — run ffmpeg with `-vf "select='gt(scene,<threshold>)',showinfo" -f null -` (write nothing); scene timestamps appear on stderr → `parseSceneCutsMs(stderr, durationMs)`.
    - `decodeFramesAt(ref: string, timesMs: number[], resolution: ResolutionTier): Promise<FrameImage[]>` — for each time, extract one frame with `ffmpeg -ss <seconds> -i <ref> -frames:v 1 -f image2pipe -vcodec png -` (or `-vcodec mjpeg`), capture stdout as a Buffer, `.toString("base64")`. Resolution policy: `low` → downscale via `-vf scale=...` (e.g. cap longest side ~512), `high` → no downscale. Set `mediaType` to match the encoder ("image/png" or "image/jpeg"). Return one FrameImage per input time, in order. Reuse the `FrameImage` type from `./assemble.js` — do not redefine it.
    - `fetchTranscript(ref: string): Promise<{ segments: TranscriptSegment[]; source: TranscriptSource | "none" }>` — BEST-EFFORT and local-first: attempt yt-dlp captions only when `ref` looks like a URL; otherwise (and on ANY failure, missing binary, or empty result) resolve to `{ segments: [], source: "none" }`. Do NOT require `whisper` (absent on this machine) or network access. Never throw out of this function. Whisper/caption parsing depth is deferred — a clean "none" fallback is a complete, acceptable implementation for this plan.

    Import `FrameImage` from `./assemble.js`; import `ResolutionTier`, `TranscriptSegment`, `TranscriptSource`, `MediaType` from `../contract/index.js`. Do not import the contract validator here.
  </action>

    `SampleOptions = { ref: string; budget?: number; resolution?: ResolutionTier; sceneThreshold?: number }`
    (default `budget` = the pure core's default ~16; default `resolution` = "low" per DESIGN §3; default `sceneThreshold` left to the effect default).

    Orchestration (explicit, sequential, all effects at this boundary):
    1. `durationMs = await probeDurationMs(ref)`.
    2. `sceneCutsMs = await detectSceneCutsMs(ref, durationMs, sceneThreshold)`.
    3. `selected = selectFrameTimes({ sceneCutsMs, durationMs, budget })` — the pure decision layer picks the final, budget-capped times.
    4. `images = await decodeFramesAt(ref, selected.map(s => s.tMs), resolution)` — decode ONLY the selected times (PETE: bounded by budget), preserving order so `images[i]` pairs with `selected[i]`.

    6. `fpsSampled = selected.length && durationMs ? selected.length / (durationMs / 1000) : 0` (effective sampled fps).
    7. `return assembleWatchedFrameSet({ ref, durationMs, fpsSampled, selected, images, resolution, transcript: segments, transcriptSource: source })`.
